# 变体评价报告模板（精简版）

> 给评审委员会看的变体观察。**目标：一屏能看完**。
> 原则详见 `SKILL.md` + `references/variant-evaluation.md`，本文件只管输出形态。

---

## 输入

- 变体截图（必填）+ `business_line`（必填）
- 业务背景：为什么做、预期解决什么问题、期望指标（业务方自述，Skill 原样保留）
- 对比样本：同场景标准卡截图（可选但强烈建议）

---

## 报告模板（四段式）

```markdown
# 变体观察 - {{YYYY-MM-DD}}

**变体**：{{一句话描述，如"推荐流新增 AI 摘要标签"}}
**业务线**：{{business_line}}
**card_type（识别）**：{{内容-图文卡 / 内容-视频卡 / 内容-直播卡 / ...}}

## 一、跨场景同物种识别 ★

| 信号 | 通过 |
|---|---|
| 宽度/间距/圆角 | ✅/❌ |
| 封面占主体 ≥ 60% | ✅/❌ |
| 标题在封面下方 | ✅/❌ |
| 有作者信息 | ✅/❌ |
| 互动数据/标签位置 | ✅/❌ |

**初判**：{{通过 / 不通过，一句话原因}}

## 二、核心观察（≤ 3 点）

1. **{{观察点 1}}**：{{体验代价 1-2 句}} | 业务可能的理由：{{1 句}}
2. **{{观察点 2}}**：{{...}}
3. **{{观察点 3，如无可省}}**：{{...}}

## 三、评分参考（不是判决）

| 项 | 分值 | 得分 |
|---|---|---|
| 同物种识别（一票否决） | 40 | {{40/0}} |
| 用户价值举证 | 20 | {{X}} |
| 与标准卡共存兼容 | 15 | {{X}} |
| 学习成本 | 10 | {{X}} |
| 视觉一致性 | 10 | {{X}} |
| 数据指标友好度 | 5 | {{X}} |
| **合计** | 100 | **{{X}}** |

## 四、讨论建议

- **给业务方**：{{一句话，需要提供什么材料或做什么调整}}
- **给规范 owner**：{{一句话，是否需要扩展规范}}
- **给评审委员会**：{{最值得讨论的核心问题，一句话}}

---

**最终决策权在评审委员会**，本报告是输入之一，不是结论。
```

---

## 关键纪律

1. **四段封顶**：识别 + 核心观察 + 评分 + 讨论建议
2. **核心观察最多 3 条**：超过 3 条说明变体太复杂，建议拆分提报
3. **每条观察最多 2 行**：体验代价 1-2 句，业务理由 1 句
4. **评分是参考，不是判决**：委员会看分数 + 看讨论建议，不直接按分数裁决
5. **讨论建议各 1 句**：给业务方 / 规范 owner / 评审委员会各一句，不展开

---

## 评分档次参考（供委员会参考，不是硬阈值）

- ≥ 80：体验代价小，建议升级为正式分支
- 60~79：有争议，建议试点 + 数据验证后再定
- < 60：体验代价明显，需要重新思考或走豁免
- **同物种识别 = 0**：身份问题最严重，先解决再谈其他
