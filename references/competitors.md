# 竞品双列卡设计对比

> **素材来源**：规范专项 v4.0 第三节《内容双列卡调研》+ web-researcher 公开资料补充。
>
> **本文件的用途**：
> - 给设计师：当设计决策被追问"为什么是这个数"时，有横向参考
> - 给规范 owner：友商迭代时快速对标，判断京东规范是否需要跟随更新
> - 给 Skill 判定：识别"这张卡更像哪家风格"——用于变体评估时的"跨场景同物种"参考

## 一、小红书（★ 内容双列卡的行业标杆）

**定位**：国内内容双列卡设计的标杆，瀑布流双列布局已成行业参考。

### 卡片结构
```
┌─────────────────────┐
│      封面图         │  ← 变高（自适应，图片不裁切）
│      (3:4 或更高)   │
│                     │
├─────────────────────┤
│ 📌 标题文字         │  ← 2 行，超出省略，加粗，黑色
│ 👤 昵称    ♥ 赞数   │  ← 灰色小字，左右分布
└─────────────────────┘
```

### 规范观察
| 维度 | 观察结果 |
|---|---|
| 布局 | 双列瀑布流，卡片高度完全自适应 |
| 图片比例 | 不裁切原图，保持原始比例（通常 3:4 ~ 2:3） |
| 卡片宽度 | (375-8-8-8)/2 ≈ 175px，8px 三边间距 |
| 卡片圆角 | 12px（较大圆角，视觉柔和） |
| 标题 | 13px，2 行限制，行高 1.4 |
| 作者区 | 头像 16px 圆形 + 昵称 + 右侧点赞数（心形图标） |
| 封面角标 | 视频卡右下角播放时长，图集右上角张数 |
| 加载方式 | 懒加载，**无骨架屏**，占位色为灰色背景 |
| 间距 | 左右各 8px 边距，中间 8px 间隔，卡片间 8px 垂直间距 |

### 设计亮点
- **不裁切图片** → 尊重创作者构图，封面质量更高
- **瀑布流** → 错落感让页面更有活力，停留时间更长
- **标题是关键钩子** → 字号不大但位置优先，用户扫读标题决定是否点击

### 京东的对齐情况
**京东专项 v4.0 明确建议"参考小红书采用自适应瀑布流"**，本 Skill 的 layout.md 和 component.md 大量参考小红书。

## 二、抖音（视频内容双列卡）

**定位**：抖音双列卡主要出现在**搜索结果页**和**同城/发现页**，与主场景竖屏全屏单列不同。

### 卡片结构
```
┌─────────────────────┐
│   视频封面帧        │  ← 固定 9:16 竖版比例
│                     │
│ ▷ 播放量            │  ← 左下角，白色文字叠加
├─────────────────────┤
│ 标题文字 2行        │
│ 👤 昵称             │
└─────────────────────┘
```

### 规范观察
| 维度 | 观察结果 |
|---|---|
| 布局 | **双列网格（等高）**，非瀑布流 |
| 图片比例 | 9:16 竖版固定（内容是竖视频） |
| 卡片高度 | 固定，左右两列对齐 |
| 封面信息 | 播放量（万）叠加左下角，白色+阴影 |
| 标题 | 2 行，13px |
| 作者信息 | **仅昵称，无头像**（信息简洁） |
| 卡片间距 | 约 4-6px（非常紧凑） |
| 交互 | 点击直接进入竖版全屏播放 |

### 设计亮点
- 竖版封面在双列中自然形成高信息密度
- 播放量叠加封面（不占卡片文字区），充分利用空间
- 从双列缩略图 → 全屏播放，有明显「模式切换感」增强沉浸

### 对京东的借鉴
- **视频卡 9:16** 直接对齐抖音（本 Skill 已采纳）
- 但京东不采用抖音的"无头像"方案（京东内容卡必须有头像+昵称）

## 三、Instagram（Explore 页）

**定位**：国际内容双列卡的代表性设计。

### 规范观察
| 维度 | 观察结果 |
|---|---|
| 布局 | **三列为主**，但 Reels 探索区为双列 |
| 图片处理 | 严格裁切为正方形（1:1），统一整齐 |
| 卡片文字 | **无文字**，纯图片/视频缩略图 |
| 视频标识 | 右上角小摄像机图标 |
| 混排规则 | 每隔约 6 格出现一个大卡（占 2 列宽） |
| 卡片间距 | 约 2-3px（极紧凑，近乎无缝） |
| 加载 | 模糊占位 → 清晰图，非常平滑 |

### 设计亮点
- 纯视觉驱动，零文字干扰 → 适合图片质量极高的平台
- 大小卡片混排节奏感打破单调，引导视线移动
- 极窄间距营造画廊感，高端氛围

### 对京东的借鉴
- **Instagram 路线不适合京东**（京东要兼顾购买转化，标题/作者/商品挂载都需要）
- 但 Instagram 的"视觉一致性极致"值得学习，避免卡片视觉过于喧闹

## 四、大众点评（生活服务内容卡）

**定位**：**生活服务 + 内容**的混合形态，代表 LBS 场景下的内容卡设计。

### 卡片结构
```
┌─────────────────────┐
│      封面图          │  ← 约 4:3，统一裁切
│                     │
├─────────────────────┤
│ 📍 商户名           │  ← 加粗，黑色
│ ☆4.8 (1.2k评)      │  ← 星级 + 评论数
│ 人均 ¥88            │  ← 价格（辅助，非主信息）
│ 标签  标签          │  ← 品类/特色标签
└─────────────────────┘
```

### 规范观察
| 维度 | 观察结果 |
|---|---|
| 布局 | 双列网格，卡片等高 |
| 图片比例 | 约 4:3，统一裁切 |
| 核心信息 | 商户名 + 评分 + 价格 |
| 评分展示 | 星星图标 + 数字 + 括号评论数 |
| 标签 | 2-3 个品类/特色标签，灰色背景小胶囊 |
| 距离信息 | 部分卡片显示距离（m/km） |
| 卡片间距 | 约 8-10px |

### 对京东的借鉴
- **点评场景的"商户名 + 评分 + 价格"**结构，对京东评价业务有直接借鉴意义
- 但京东评价卡是"用户视角"（基于购买体验），点评是"商户视角"——信息层级主语不同

## 五、美团闪购/超市（服务内容卡）

**定位**：**服务 + 内容**混合形态，出现在商超、闪购、榜单场景。

### 规范观察
| 维度 | 观察结果 |
|---|---|
| 布局 | 双列网格，等高 |
| 图片比例 | 约 3:2，有时 1:1 |
| 核心信息 | 商家名 + 优惠 + 起送价/配送费 |
| 配送时效 | "预计 XX 分钟"置于封面角标 |
| 评分 | 数字评分（非星星），简洁 |
| 活动标签 | 红色角标叠加封面（"新客减 X"等） |

### 对京东的借鉴
- 有限借鉴 —— 美团偏服务属性，和京东内容卡的"好奇心/共鸣感"驱动不同
- 但**"预计 XX 分钟"角标**的"信息置于封面但不打扰"做法值得参考

|---|---|---|---|---|---|---|---|---|

## 八、对京东的设计启示（v4.0 原文）

1. **图片比例**：统一使用 **3:4 竖图**，符合移动端内容消费习惯（占据更多垂直空间）
2. **布局选择**：**瀑布流 + 自适应高度**，突出内容差异化
3. **文字展示**：**极简策略，最多 2 行**，突出视觉内容
4. **标签处理**：话题/地点标签**贴图片底部**，点击无跳转（浮层交互）
5. **视频体验**：支持**自动播放（静音）**，点击全屏

## 九、友商迭代跟踪机制

按 v4.0 风险应对："友商快速迭代导致规范滞后 → **每季度友商调研**，规范小版本更新"。

**本文件的更新节奏**：
- **每季度**：friend-researcher 跑一次友商最新版本调研，更新本文件
- **友商大改版时**（如小红书 UI 大改）：触发紧急调研，评估是否需要京东规范跟随

每次更新在 `changelog.md` 留痕，包含：
- 新增观察
- 变化点（和上一版对比）
- 是否建议京东规范调整
