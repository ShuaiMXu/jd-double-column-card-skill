### 2026-04-14（v0.5.2 — 首次真实测试驱动的两个原则级修订）

Shaka 用一张点评场域页面截图做实战测试，暴露出 v0.5.1 的两个原则级问题。本次修订**非小补丁，是基础原则调整**。

#### 问题 1：家族识别把动态数据当结构信号（错）

**反例**：v0.5.1 决策树写 `IF 标题下有"X 万人种草" → 点评，ELSE → 内容`。
**真实情况**：种草数是运行时数据，**0 人种草就不显示**。用"有无种草数"识别家族 → 错把"未激活的点评卡"判成"内容家族卡"。

**修正**：
- 内容家族和点评家族**底部基础结构相同**（头像+昵称+互动数据位）
- **不能只靠底部区分**，必须用 `business_line` 辅助
- 种草数 / 身份标签 / 观看量等降级为"**激活信号**"（记录 annotations，不参与家族判定）

**影响文件**（决策树同步修订）：
- `site-map.md` 第三节 → 分两层 Layer 1+2
- `business-baselines.md` 第四节 → 分两层 Layer 1+2
- `prompts/page-level-review.md` Step 2a → 完整修订
- `SKILL.md` → 新增**原则 8：动态数据不作家族识别硬条件**

#### 问题 2：Skill 错误地把用户上传内容纳入观察范围

**反例**：v0.5.1 报告里写 "Card X 酒店图片偏横 → 瀑布流节奏不齐"。
**真实情况**：封面内容是用户/运营上传，**设计师无法强控制**。规范只能管"框"，不能管"画里的内容"。

**修正**：
- 明确 Skill 观察范围 = **设计师规范可控的结构性元素**
- 不观察 = **用户/运营上传的内容层**（封面具体内容、比例偏向、色彩、标题文案质量等）
- 例外：图片比例超出规范值域（如 1:2）属值域违规，要报

**影响文件**：
- `SKILL.md` → 新增**原则 7：结构优先、内容无涉（Structure over Content）**
- `prompts/page-level-review.md` Step 4 → 新增执行约束 + 典型错误示例

#### v0.5.2 新增两条核心原则

- **原则 7：结构优先、内容无涉** —— Skill 只看设计师可控的结构，不看内容层
- **原则 8：动态数据不作家族识别硬条件** —— 家族识别靠结构不变量，动态数据只做 annotations

#### 动作清单

| 动作 | 文件 | 内容 |
|---|---|---|
| 新增 | `SKILL.md` | 原则 7 + 原则 8 + 版本 v0.5.2 |
| 修订决策树 | `site-map.md` 第三节 | 两层识别（基础结构 + business_line 辅助）+ 动态数据速查表 |
| 修订决策树 | `business-baselines.md` 第四节 | 同步修订 |
| 修订决策树 + 新增执行约束 | `prompts/page-level-review.md` | Step 2a 决策树 + Step 4 结构优先约束 + 典型错误示例 |

#### 测试结果（v0.5.2 前的错误 + 修正后的正确）

**测试输入**：点评场域页面截图（4 张完整卡 + 2 张半截卡）

| 卡 | v0.5.1 识别（错） | v0.5.2 识别（对） |
|---|---|---|
| Card 1（烤鸭控） | 内容家族（理由：无种草数）| 点评家族（种草数为 0，annotations.has_seed_count=false）|
| Card 2（水饺求真官） | 点评·身份标签变体 | 同（不变） |
| Card 3（铜锅涮肉） | 点评 | 同（不变）|
| Card 4（酒店） | 点评 + "图偏横"误报差异 | 点评（删除"图偏横"误报）|

**差异观察从 3 条降到 2 条**（删除了内容层伪差异）。

### 2026-04-14（v0.4.0 — 五大家族体系建立 + 22 个基线沉淀）

Shaka 提供了 22 张代表性卡片样式，逐张标注业务线/家族归属，让 Skill 完成了一次**架构级升级**。

**关键认知突破**：
- 京东双列卡不是二元（内容 vs 商品）而是 **五元家族**：内容 / 商品 / 圈子 / 点评 / 新品
- 每个家族有自己的 **底部不变量（DNA）**，**家族身份由底部决定，不由封面决定**
- "顶部+中部统一识别语言、底部锚定家族身份" = 京东双列卡的设计哲学

**采集成果（写入 `business-baselines.md`）**：

| 家族 | 已采子类型 |
|---|---|
| 内容 | 图文（点赞/观看互动）+ 直播（基础/红包雨/超头/预约/沉浸式）共 7 |
| 商品 | 视频卡 + 直播卡（基础/弹幕外露/沉浸式商卡/沉浸式促销）共 5 |
| 圈子 | 图文卡（视频卡按规律预测）共 1 |
| 点评 | 图文/视频 + 种草混血 *2 共 4 |
| 新品 | 入口 / 直播入口 / 测评视频 / 趋势 / 明星同款 共 5 |

**8 条跨家族规约**（沉淀为 L1 大规范候选）：
- L1-1 左上角 = 卡片属性表达位
- L1-2 右上角 = 辅助信号位（可为空）
- L1-3 标题下方 = 元信息位（视觉权重 ∝ 信号公共性）
- L1-4 商品挂载浮条位置 = 反映叙事方向（封面下/标题下两种）
- L1-5 **底部不变量 = 家族 DNA 锚定**（最核心）
- L1-6 内容卡右下互动信号位可切换
- L1-7 封面浮层 = 决策信息前置位（描述性，不强制）
- L1-8 梯度叠加封面 = 一种封面结构类型（描述性）

**文件操作**：
| 动作 | 文件 | 内容 |
|---|---|---|
| 重写 | `business-baselines.md` | 22 个基线 + 五大家族体系 + 8 条规约 + 家族识别决策树 + 反哺其他文件清单 |
| 留待 v0.4.1 | `layout.md` / `component.md` / `site-map.md` / `SKILL.md` / `prompts/review-screenshots.md` | 按 baselines 第七节清单修订（家族扩展 + L1 规约升级） |

**关键设计哲学沉淀**：
- "**家族身份由底部决定，不由封面决定**" —— 翻转对照（沉浸式·达人 vs 沉浸式·商品）的实证
- "**新品家族 5 子类型 = 购买决策 5 阶段**" —— 造势 → 参与 → 验证 → 探索 → 共鸣
- "**明星 IP = 隐形作者**" —— 在规约内创新的典范
- "**视觉权重 ∝ 信号公共性**" —— 公共信号强视觉，个人化信号弱视觉

**Skill 自身能力边界教训**：
- Shaka 的"Skill 识别深度匹配规范深度"指示 —— 业务方没规范的地方（梯度叠加每层细节、明星同款的具体趋势元素）Skill 不下钻

**采集尚缺**：

- 内容直播观看数情况
