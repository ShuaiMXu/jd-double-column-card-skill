# 业务线卡片基线（Business Baselines）

> 京东双列卡的**家族体系 + 基线样式 + 跨家族设计规约**。
>
> **本文件用途**：Skill 做观察时三角对比的第二参照物（第一是 `layout.md` 的 L1 大规范，第三是线上实际样本）。
>
> **维护协议**：新基线由业务方提供截图 + Shaka 确认后追加。非 Shaka 确认不写入。
>
> **版本**：v0.2（2026-04-14 第二轮整合，22 个基线 + 完整家族体系）

---

## 一、核心架构：京东双列卡的"五大家族"

京东 App 是多业务线共生的超级平台。每个业务线有自己的"家族 DNA"，**家族身份由"底部不变量"锚定**。

### 五大家族总览

| 家族 | 心理锚点 | 底部不变量（DNA） | 卡片"主语" |
|---|---|---|---|
| **内容** | UGC 信任 + 社交证明 | 头像 + 昵称 + 互动数据 | 创作者（人） |
| **商品** | 商家背书 + 销量证明 | 店铺 logo + 店铺名 + 销售数据 | 商家 |
| **圈子** | 集体归属感 | 圈子图标 + 圈子名（无作者无互动） | 社群 |
| **点评** | UGC 信任 + 决策证明 | 头像 + 昵称 + 互动 + 标题下种草数 | 推荐者（人）+ 决策证明 |
| **新品** | 事件 / 活动 / 趋势驱动 | 按子类型变化（CTA 按钮 / 事件胶囊 / 商品挂载 / 话题标签等）| **抹除主语**（聚焦事件本身） |

### 家族体系的设计哲学

**"顶部 + 中部统一识别语言；底部锚定家族身份"**：
- **封面 + 顶部**：跨家族共享识别语言（属性表达位、辅助信号位、元信息位）→ 用户在站内任意业务线之间跳转，识别心智可复用
- **底部**：保留家族 DNA 差异 → 业务线表达独特叙事
- **效果**：多业务线 → 统一 App 心智

## 二、跨家族设计规约（Cross-Family Conventions）

> 这是本文件最有价值的产出。从 22 个基线里提炼出的站内统一规约。
>
> 标记 **[L1 级]** 的建议升级到 `layout.md` 大规范，作为所有卡的硬性约束。

### L1-1：左上角 = 卡片属性表达位

**作用**：表达"非默认媒介属性"。

| 属性 | 左上角内容 |
|---|---|
| **视频**（非默认） | 视频角标（白色播放 ▶，部分场景带时长） |
| **直播**（非默认） | 红色直播图标 + 观看数（视场景） |
| **动图**（未来）| 动图 icon |
| **图文**（默认态）| **大多数场景为空**；个人页场景显式标图文 icon |

**附加规则**：左上角以属性 icon 为锚点，可**共生其他识别性信号**：
- 话题标签（如种草卡的"#XXX >"）
- 浏览历史提示（"刚刚看过"）
- 用户状态（"我的关注"）

### L1-2：右上角 = 辅助信号位（位置固定、内容可变、可为空）

| 卡型 | 右上角内容 |
|---|---|
| 商品直播卡 | 店铺胶囊 |
| 内容直播卡·红包雨变体 | 活动胶囊（红包雨/抽奖等） |
| 其他卡型 | 大多为空 |

### L1-3：标题下方 = 关于"内容"的元信息位

承载关于**内容本身**的元信息。**视觉权重 ∝ 信号公共性**：

| 信号类型 | 案例 | 视觉权重 |
|---|---|---|
| 公共社交证明 | "12.3 万人种草"（点评）/ "服饰美妆排行榜第1名"（达人榜单）| 中-强 |
| 用户状态（个人化）| "你预约的" / "你关注的"  | **弱**（灰色小字）|
| 紧凑数字（同行右侧）| "8.5 万"（观看量）| 弱-中 |

**设计哲学**：公共信号要被所有人看到（强视觉），个人化信号只对你有意义（弱视觉，记忆唤起即可）。

**与 L1-9 的关系**：本规约处理"关于内容的元信息"，**关于"人"的元信息走 L1-9（昵称旁）**。位置 = 主语的视觉锚点。

### L1-4：商品挂载浮条的位置 = 反映叙事方向

之前 component.md 写的"底部小浮条"**不准确**。线上实际有两种位置，对应两种叙事方向：

| 浮条位置 | 案例 | 叙事方向 |
|---|---|---|
| **封面下、标题上**（灰底）| 种草卡 | 商品是**起点**（商品 → 内容是商品的延伸） |
| **标题下、最底部**（白底）| 新品测评视频卡 / 明星同款卡 | 商品是**终点**（看内容 → 决定是否点商品） |

### L1-5：底部不变量 = 家族 DNA 锚定（最核心规约）

> **家族身份由底部决定，不由封面决定**。
>
> 封面可以任意变化（基础款 / 梯度叠加 / 沉浸式场景），底部不变 → 家族不变。

**实证**：本轮采集中"内容·达人沉浸式"和"商品·促销沉浸式"两张卡封面相似度极高，但底部不同 → 家族完全不同。**用户 0.5 秒扫视时锚点是底部，不是封面**。

### L1-6：内容卡家族的"右下互动信号位"可切换

内容卡右下的互动数据图标 + 数字组合**可按内容性质切换**：

| 子样式 | 右下图标 | 用例 |
|---|---|---|
| 点赞互动（基础）| ❤️ | 偏社交内容 |
| 观看量互动 | 👁 | 偏视频/消费内容 |
| 评论互动 | 💬 | 偏深度讨论 |
| 收藏互动 | ⭐ | 偏工具/收藏型 |

**骨架固定、信号可选** —— 业务线表达自由度在"右下信号位"。

### L1-7：封面浮层 = 决策信息前置位（描述性，不强制规约）

**部分卡**采用封面叠加附加信息层。位置相对自由，**视觉权重要克制不能完全遮盖封面**。具体规则由业务方决定，Skill 不下钻细节。

| 浮层内容类型 | 典型案例 |
|---|---|
| 销量证明 | "本场热卖 ×286" |
| 模拟弹幕 | "星火·天冷，求保暖" |
| 关键属性标签 | "麂皮面料"（趋势卡）|
| 沉浸式商品卡 | 白底商品缩略 + 价格 |
| 活动叠字 | "618 采销降价火热进行中" |

### L1-8：梯度叠加封面 = 一种封面结构类型（描述性）

封面图本身是**多层图层叠加**（场景图 → 白底图 → 直播间封面合成）。**Skill 识别"这是梯度叠加封面"即可，不深挖每层规则**。

### L1-9：昵称旁 = 关于"人"的元信息位 ★ v0.4.1 新增

承载用户认证身份等关于**人本身**的元信息。位置：紧贴昵称右侧。

| 内容类型 | 案例 | 视觉风格 |
|---|---|---|
| 用户认证身份 | "求真官"（点评业务） | 红色小胶囊 |
| （未来可能） | 达人 / 品牌官 / V 标 / 专家 | 待采样确认 |

**设计哲学（与 L1-3 共同支撑的更高规则）**：

> **元信息按"主语"分位置，位置 = 主语的视觉锚点**
>
> - 陈述**内容**（这内容怎么样）→ **标题下方**（紧贴内容描述）
> - 陈述**人**（这人是谁）→ **昵称旁**（紧贴人）
>
> 用户想了解"谁说的"，眼睛会自然落到说话人附近；想了解"内容怎么样"，眼睛会落到内容附近。**位置不是布局选择，是用户视线路径的设计**。

## 三、家族基线详解

### 3.1 内容家族（Content）

#### 3.1.1 内容-图文卡·基础样式

```
【业务线】推荐 / 其他内容流（个人页除外）
【封面】3:4 竖图，不裁切，干净
【左上角】空（图文是默认态）
【必含】
  - 封面图
  - 标题（加粗黑字，1-2 行，允许内嵌 emoji）
  - 头像 + 昵称（左下）
  - 互动数据（右下，图标 + 数字）
【不含】封面叠字 / 话题标签 / 价格
```

#### 3.1.2 互动模式变体（"右下信号位"切换）

基础结构相同，仅切换右下图标：

| 子样式 | 右下图标 | 数据 |
|---|---|---|
| 点赞互动（基础） | ❤️ | 点赞数 |
| 观看量互动 | 👁 | 观看量 |
| （预留）评论互动 | 💬 | 评论数 |
| （预留）收藏互动 | ⭐ | 收藏数 |

#### 3.1.3 内容-直播卡·基础样式

```
【业务线】直播（内容叙事路线）
【封面】3:4 竖图，可含动态画面 + 叠字作为内容设计
【左上角】红色直播图标
【必含】
  - 封面图
  - 标题（加粗，1-2 行）
  - 头像 + 昵称（左下）
  - 互动数据（右下）
【家族归属】内容家族（底部锚定）
```

#### 3.1.4 内容-直播卡·变体（在基础结构上叠加）

| 变体 | 新增元素 | 位置 |
|---|---|---|
| **红包雨变体** | 活动胶囊（"5000元 红包雨"） | 右上角 |
| **超头/达人变体** | 榜单/认证（"服饰美妆排行榜第1名"） | 标题下方（公共信号，中视觉） |
| **预约变体** | 用户状态（"你预约的"） | 标题下方（个人化，弱视觉灰色小字） |
| **沉浸式（达人/超头）变体** | 沉浸式场景摄影 + 活动叠字 + 左中"我的关注"胶囊 | 封面整体重构 |

**变体可叠加组合**（如超头 + 红包雨 + 预约可同时存在）。

### 3.2 商品家族（Product）

#### 3.2.1 商品-视频卡

```
【封面】静态产品图，约 3:4 ~ 1:1，白色/干净背景
【左上角】视频角标
【必含】
  - 封面图
  - 商品名标题（加粗）
  - 价格（"¥XX 到手价"，加粗）
【可选】促销标签（满减/运费险等，灰色胶囊）
【不含】作者头像/昵称、时长标识（待确认是否所有商品视频卡都无）
```

#### 3.2.2 商品-直播卡·基础样式

```
【封面】9:16 竖版，商品场景图（允许人物模特）
【左上角】红色图标 + 观看数（如"1.2 万观看"）
【右上角】店铺/品牌名（白色胶囊）
【必含】
  - 封面图
  - 商品名标题
  - 价格（红色加粗）
【可选】
  - 封面叠字：上部直播预告 + 中部商品说明
  - 促销信息（满减/福利）
  - 品牌 logo 胶囊（左下角）
  - 辅助媒介信号（麦克风 icon 等）
【家族归属】商品家族（底部锚定店铺名）
```

#### 3.2.3 商品-直播卡·变体

| 变体 | 关键差异 |
|---|---|
| **决策信息前置变体（弹幕外露）** | 封面叠加多层决策信号（销量浮条 + 模拟弹幕 + 浏览历史"刚刚看过"） |
| **沉浸式商卡（场景图）变体** | 封面用真实使用场景摄影 + 白底商品卡浮层 |
| **沉浸式（促销氛围）变体（翻转版）** | 封面用直播间真实氛围 + 活动胶囊 + 弹幕气泡（与"内容·达人沉浸式"形成翻转对照） |
| **梯度叠加封面变体** | 封面是多层图层合成（场景 → 白底 → 直播封面）；Skill 识别即可不深挖 |

### 3.3 圈子家族（Community）

#### 3.3.1 圈子-图文卡

```
【业务线】圈子
【封面】3:4 竖图，干净
【左上角】空（默认态）
【必含】
  - 封面图
  - 标题（加粗，1-2 行）
  - 圈子图标 + 圈子名（左下）
【不含】
  - 作者头像/昵称
  - 互动数据（右下空）
【家族 DNA】只锚定"圈子归属"，不展示作者/互动
```

#### 3.3.2 圈子-视频卡（按规律预测，待确认）

```
【基础结构】继承「圈子-图文卡」
【唯一差异】左上角加视频角标
```

**圈子家族的体验哲学**：圈子先于内容存在，用户对**圈子**有认同感而非单条内容。卡片只是"通往圈子的门"——门上贴的是圈子名而非作者。

### 3.4 点评家族（Review）

#### 3.4.1 点评-图文卡

```
【业务线】点评
【封面】3:4 竖图，干净
【左上角】空（默认态，非个人页）
【必含】（推荐内容图文卡基础结构 +）
  - ★ 种草数行（绿色 ✓ + "X 万人种草"，标题下方左侧）
  - 头像 + 昵称（左下）
  - 互动数据（右下）
【点评特有信号层】
  - 种草数承担"购前决策社交证明"叙事
```

#### 3.4.2 点评-视频卡

```
【基础结构】完全继承「点评-图文卡」
【唯一差异】左上角加视频角标
```

#### 3.4.3 点评-图文卡·用户身份标签变体 ★ v0.4.1 新增

```
【基础结构】继承「点评-图文卡」

【新增/差异】
  - 昵称右侧：用户身份标签（如"求真官"红色胶囊）
  - 封面叠字钩子（如"谁家好人会点北京烤鸭外卖啊"红色横幅）

【可与种草数并存】
  - 标题下方"X 万人种草" = 关于内容的认可（走 L1-3）
  - 昵称旁"求真官" = 关于用户的认证（走 L1-9）
  - 两者位置不冲突，可同时使用

【家族归属】
  仍属点评家族，身份标签是底部 DNA 的"附加层"，不替代基础结构
```

**这张卡是 L1-9 规约的实证来源。**

#### 3.4.4 种草-图文卡 / 种草-视频卡（个人页混血）

> 种草卡是**"内容+商品"混血体**。归类讨论：可作为"内容家族·种草子分支"或"独立混血家族"，本基线放在点评家族附近作为参考。

```
【业务线】个人中心（种草场景）
【封面】3:4 竖图

【必含】
  - 封面图
  - ★ 左上角：话题标签（蓝字 hashtag + 右箭头）+ 图文 icon（共生在"个人页"才显示 icon）
  - ★ 商品挂载浮条（封面下、标题上，灰底）：价格 + 划线原价 + "已售 X 万+"
  - 标题（加粗，1-2 行）
  - 头像 + 昵称（左下）
  - 互动数据（右下）

【种草视频卡的差异】
  - 左上角媒介 icon：图文 icon → 视频 icon
```

### 3.5 新品家族（New Arrival）★ 唯一抹除主语的家族

新品家族**不展示作者/店铺/圈子**——卡片的存在理由是某个事件/活动/趋势/测评，**而不是某个人或某个商家**。

新品家族用 5 个子类型完整覆盖**电商购买决策的五大心理阶段**：

| 子类型 | 决策阶段 | 核心驱动 | 底部 DNA |
|---|---|---|---|
| 入口卡 | **造势** | 时间感 + IP | CTA + 装饰 |
| 直播卡入口 | **参与** | 围观感 | 事件胶囊 + 互动激励 |
| 测评视频卡 | **验证** | 测评数据 | 商品挂载浮条 |
| 趋势专属图文卡 | **探索** | 趋势/审美 | 话题标签 + 热度 |
| 明星同款卡 | **共鸣** | 明星 IP | 商品挂载 + 种草数 |

#### 3.5.1 新品-入口卡

```
【封面】3:4，IP 联名场景
【左上角】红色胶囊 "新发布 + MM/DD"（时间感 = 核心识别）
【必含】
  - 封面图
  - 标题（商品名）
  - 底部：CTA 按钮（"XX 限定" / "立即查看"）
  - 底部：装饰 IP 图（如蜘蛛侠人偶）
```

#### 3.5.2 新品-直播卡入口

```
【封面】3:4，发布会/新品场景
【左上角】红色直播图标 + 观看数（"X 万人围观"）
【必含】
  - 封面图
  - 标题（发布会名，如"XX 新品发布会"）
  - 底部左：白色胶囊 "📷 重磅发布"（事件标识 + 摄影机 icon）
  - 底部右：互动激励 icon 组（送礼/点赞/收藏预告）
```

#### 3.5.3 新品-测评视频卡

```
【封面】3:4，使用场景 / 测评场景
【左上角】视频角标 + 时长（"00:10"，★ 测评视频时长必含）
【必含】
  - 封面图
  - 标题（测评内容名）
  - 标题右侧或下方：观看量数字
  - 底部：白底商品挂载浮条（产品缩略 + 商品名 + 卖点描述）
```

#### 3.5.4 新品-趋势专属图文卡

```
【封面】3:4，多图拼图集锦（趋势合辑）
【封面浮层】关键趋势元素标签（如"麂皮面料"、"琥珀色针织面料"）
【左上角】空
【必含】
  - 封面图
  - 底部：话题标签（"#流光复古风"）+ 话题热度（"⭕ 5000"）
```

#### 3.5.5 新品-明星同款卡

```
【封面】3:4，明星生活照/街拍（明星本人 = 隐形作者）
【左上角】空
【必含】
  - 封面图（明星本人形象作为视觉主体）
  - 标题（"明星名 + 内容主题"，如"Jennie机场出发Top私服穿搭合集"）
  - 标题下方：观看量
  - 底部：商品挂载浮条 = 商品名（如"Chanel 春夏外套"）+ 种草数（"9999 人种草"，红字加粗）

【设计巧思】
  明星 IP 不在底部展示头像昵称（守住新品家族 DNA），
  而是用封面承担"作者/IP"视觉权重 → "在规约内创新"的典范
```

## 四、家族识别决策树（给 Skill 用，v0.5.2 修订）

**关键修订**：家族识别只依赖**结构不变量**，**不依赖动态数据**（种草数、观看量等运行时可能为 0 → 不显示的元素）。

```
看到一张双列卡 →

Step 1: 看底部基础结构

CASE A: 底部 = 头像 + 昵称 + 互动数据
    # 内容家族和点评家族结构相同，靠 business_line 辅助
    IF business_line ∈ {点评, 个人中心}: → 点评
    ELSE IF business_line = 推荐:       → 内容
    ELSE:                                → 模糊（低置信度）

    # 种草数 / 身份标签 / 互动图标 作为"激活信号"记录
    # 不影响家族判定（种草数 = 0 时就不显示）

CASE B: 底部 = 店铺 logo + 店铺名 + 销售数据    → 商品

CASE C: 底部 = 圈子图标 + 圈子名（右下为空）      → 圈子

CASE D: 底部 = [CTA按钮 / 事件胶囊 / 商品挂载 / 话题标签]
        AND 无作者/店铺/圈子              → 新品

Step 2: 看封面 + 标题 → 识别子类型
Step 3: 看左上角 → 识别媒介属性
```

**核心规则**：
- 底部**结构**决定家族（不是底部具体**数据**）
- 种草数是数据层，有则显示，无则不显示 —— **不作识别条件**
- 内容家族 vs 点评家族结构重合，**靠 business_line 辅助**（这是 v0.5.2 前决策树里的 bug）
