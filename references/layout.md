## 一、基础栅格（375px 标准屏）

| 元素 | 规范值 | 说明 |
|---|---|---|
| 屏幕宽度 | 375px | 标准移动端参考屏 |
| 左右外边距 | 8px | 卡片离屏幕边缘 |
| **卡片宽度** | **176dp（固定）** | v4.0 权威值 |
| 卡片中间间距 | 8px | 左右列卡片间隔 |
| 卡片垂直间距 | 8px | 上下卡片间隔 |
| 卡片圆角 | 12px | 较大圆角，视觉柔和 |
| **左右列高度差** | **≤ 40px** | ★ 硬约束 |

**判定规则**：
- `cardWidth ∈ [174, 178]`（±2dp 容差）
- `gutter = 8px`（严格等于）
- `radius = 12px`（严格等于）
- `heightDiff ≤ 40px`（超过即差异）

## 二、L1 规约（9 条跨家族通用规则）

### L1-1：左上角 = 卡片属性表达位

**作用**：表达"非默认媒介属性"。

| 属性 | 左上角内容 | 备注 |
|---|---|---|
| 视频（非默认） | 视频角标（白色播放 ▶）| 测评视频额外带时长"00:10" |
| 直播（非默认） | 红色直播图标 | 视场景可加观看数（"1.2 万观看"）|
| 动图（未来）| 动图 icon | 规范预留 |
| 图文（默认态）| 大多数场景**为空** | 个人页场景显式标图文 icon |

**附加规则**：左上角以属性 icon 为锚点，可**共生其他识别性信号**：
- 话题标签（如种草卡 `#XXX >`）
- 浏览历史提示（"刚刚看过"）
- 用户状态（"我的关注"）

### L1-2：右上角 = 辅助信号位（位置固定、内容可变、可为空）

| 卡型 | 右上角内容 |
|---|---|
| 商品直播卡 | 店铺胶囊（如"蕉内凉皮"）|
| 内容直播卡·红包雨变体 | 活动胶囊（"5000元 红包雨"）|
| 其他卡型 | 大多为空 |

### L1-3：标题下方 = 关于"内容"的元信息位

承载关于**内容本身**的元信息。**视觉权重 ∝ 信号公共性**：

| 信号类型 | 案例 | 视觉权重 |
|---|---|---|
| 公共社交证明 | "12.3 万人种草"（点评）/ "服饰美妆排行榜第1名"（达人榜单）| 中-强 |
| 用户状态（个人化） | "你预约的" / "你关注的" | **弱**（灰色小字）|
| 紧凑数字（同行右侧）| "8.5 万"（观看量）| 弱-中 |

**设计哲学**：公共信号要被所有人看到（强视觉），个人化信号只对你有意义（弱视觉，记忆唤起即可）。

**与 L1-9 的关系**：本规约处理"关于内容的元信息"，关于"人"的元信息走 L1-9（昵称旁）。

### L1-4：商品挂载浮条的位置 = 反映叙事方向

两种位置对应两种叙事方向：

| 浮条位置 | 案例 | 叙事方向 |
|---|---|---|
| 封面下、标题上（灰底）| 种草卡 | 商品是**起点**（商品 → 内容是商品的延伸） |
| 标题下、最底部（白底）| 新品测评视频卡 / 明星同款卡 | 商品是**终点**（看内容 → 决定是否点商品） |

### L1-5：底部不变量 = 家族 DNA 锚定（★ 最核心规约）

> **家族身份由底部决定，不由封面决定**。
> 封面可以任意变化（基础款 / 梯度叠加 / 沉浸式场景），底部不变 → 家族不变。

| 家族 | 底部不变量 |
|---|---|
| 内容 | 头像 + 昵称 + 互动数据 |
| 商品 | 店铺 logo + 店铺名 + 销售数据 |
| 圈子 | 圈子图标 + 圈子名（无作者无互动）|
| 点评 | 头像 + 昵称 + 互动 + 标题下种草数（+ 可选昵称旁身份标签） |
| 新品 | 按子类型变化（CTA / 事件胶囊 / 挂载浮条 / 话题标签）| 

**识别顺序**：**先看底部定家族，再看封面定子类型，再看左上角定媒介**。

### L1-6：内容卡右下互动信号位可切换

内容家族的右下互动数据图标 + 数字组合**可按内容性质切换**：

| 子样式 | 右下图标 | 适用场景 |
|---|---|---|
| 点赞互动（基础）| ❤️ | 偏社交内容 |
| 观看量互动 | 👁 | 偏视频/消费内容 |
| 评论互动 | 💬 | 偏深度讨论 |
| 收藏互动 | ⭐ | 偏工具/收藏型 |

**骨架固定、信号可选** —— 业务线自由度在此位。

### L1-7：封面浮层 = 决策信息前置位（描述性规约）

**部分卡**在封面上叠加附加信息层。具体规则由业务方决定，Skill 不下钻细节。

| 浮层内容类型 | 典型案例 |
|---|---|
| 销量证明 | "本场热卖 ×286" |
| 模拟弹幕 | "星火·天冷，求保暖" |
| 关键属性标签 | "麂皮面料"（趋势卡） |
| 沉浸式商品卡 | 白底商品缩略 + 价格 |
| 活动叠字 | "618 采销降价火热进行中" |

**Skill 识别深度**：识别"这张卡有封面浮层"即可，不拆解每层规则。

### L1-8：梯度叠加封面 = 一种封面结构类型（描述性规约）

封面图本身是**多层图层叠加**（场景图 → 白底图 → 直播间封面合成）。

**Skill 识别深度**：识别"这是梯度叠加封面"即可，不深挖每层规则。

### L1-9：昵称旁 = 关于"人"的元信息位

承载用户认证身份等关于**人本身**的元信息。位置：紧贴昵称右侧。

| 内容类型 | 案例 | 视觉风格 |
|---|---|---|
| 用户认证身份 | "求真官"（点评业务）| 红色小胶囊 |
| （未来）| 达人 / 品牌官 / V 标 / 专家 | 待采样确认 |

**与 L1-3 共同支撑的更高规则**：
> **元信息按"主语"分位置，位置 = 主语的视觉锚点**
> - 陈述内容 → 标题下方（L1-3）
> - 陈述人 → 昵称旁（L1-9）

## 三、图片规范

| 类型 | 比例 | 处理方式 |
|---|---|---|
| 图文默认 | **3:4** | 竖图，占用更多垂直空间 |
| 视频封面 | **9:16** | 竖版全屏视频 |
| 其他可选 | 4:3 / 1:1 | 根据内容类型调整 |

**核心原则（对齐小红书）**：**不裁切图片，保持原始比例，瀑布流自适应高度**。

**判定规则**：
- 图文卡：`ratio ∈ {3:4, 4:3, 1:1}`，默认 3:4
- 视频卡：`ratio = 9:16`，必须标配左上角视频角标
- **禁止**：同一场景混排 ≥2 种比例（除非有明确业务标识）
- 懒加载 + 无骨架屏 + 灰色占位色（对齐小红书的"无闪烁"体验）

## 四、文字规范

| 元素 | 字号 | 行高 | 行数限制 | 颜色/字重 |
|---|---|---|---|---|
| 标题 | 13px | 1.4 | ≤ 2 行，超出省略 | 黑色，加粗 |
| 昵称 | 12px | 1.4 | 1 行 | 灰色 |
| 互动数据 | 11px | 1.4 | 1 行 | 灰色，右侧分布 |
| 标签 | 11px | 1.2 | 单行胶囊 | 场景色 |

**判定规则**：
- 标题长度 ≤ 24 字（中文）
- **标题必须加粗** —— 识别"本卡是内容卡而非纯图卡"的关键信号
- 标题 1 行留白在内容卡允许
- 字号严格 ±0.5px 浮动

## 五、间距与值域（2px 步长，与 NutUI token 对齐）

- **合法间距值域**：`{2, 4, 6, 8, 10, 12, 16, 20}`
- **合法圆角值域**：`{0, 4, 8, 12, 16, 18}`
- **合法字号值域**：`{10, 11, 12, 13, 14, 16, 18, 20, 24, 28}`

**判定**：值域外的数值（如圆角 14px / 间距 7px）**即差异**，无需业务论证。

## 七、常见差异模式（给 VLM 识别参考）

按优先级从高到低：

| 优先级 | 差异类型 | 描述 |
|---|---|---|
| 高 | 宽度异常 | `cardWidth < 174dp` 或 `> 178dp` |
| 高 | 左右列高度差过大 | 底部偏差 `> 40px` |
| 高 | 圆角错误 | 非 12px |
| 高 | 间距错误 | 非 8px |
| 中 | 图片裁切 | 违反"不裁切"原则 |
| 中 | 比例混排 | 同场景 ≥2 种比例无业务标识 |
| 中 | 标题溢出 | `> 2 行`未省略 |
| 中 | 标题未加粗 | 字重 `< 600` |
| 低 | 字号拍脑袋 | 值域外字号 |
| 低 | 颜色偏离 | 非 token 色 |

**注意**：这里用"优先级"而非"P0/P1/P2"，目的是**不作违规判决，作为差异观察时的关注度排序**。
